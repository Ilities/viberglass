## Becoming a Maintainer

We welcome contributions from the community. Active contributors who demonstrate consistent commitment to the project may be invited to become maintainers.

### Expectations

- Review pull requests in a timely manner
- Help triage issues and bug reports
- Contribute to project documentation
- Participate in project discussions
- Uphold the [Code of Conduct](CODE_OF_CONDUCT.md)

## Contact

For questions about the project, please:
- Open a [GitHub Discussion](https://github.com/Ilities/viberglass/discussions)
- Email: maintainers@viberglass.io

## Decision Making

Project decisions are made through consensus among maintainers. For significant changes that affect the project direction, maintainers will discuss and vote on proposals.

### Voting

- Simple majority for routine changes
- Unanimous consent for breaking changes or new major features
