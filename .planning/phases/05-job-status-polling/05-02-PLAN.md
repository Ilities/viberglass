   - Import toast from 'sonner'

2. Function signature: `useJobStatus(jobId: string)`

3. Internal state:
   - previousStatus: useState<string | null>(null)
   - Tracks status changes to detect when to show toast

4. Call usePolling with options:
   - fn: () => getJob(jobId)
   - interval: 3000 (3 seconds)
   - immediate: true
   - enabled: !!jobId (only poll if jobId exists)
   - onComplete: (job) => {
       - Check if status is terminal: job.status === 'completed' || job.status === 'failed'
       - Compare previousStatus with current job.status
       - If status changed AND previousStatus was not null:
         - For 'completed': toast.success('Job completed successfully', {
             description: 'Your changes have been applied',
             action: job.result?.pullRequestUrl ? {
               label: 'View PR',
               onClick: () => window.open(job.result!.pullRequestUrl!, '_blank')
             } : undefined
           })
         - For 'failed': toast.error('Job failed', {
             description: job.result?.errorMessage || job.failedReason || 'An error occurred'
           })
       - Update previousStatus with setPreviousStatus(job.status)
       - Return isTerminal to stop polling
     }

5. Return object: {
     job: data,
     isLoading: !data && !error,
     error,
     isPolling,
     refetch
   }

Do NOT show toast on initial mount (only when status changes from previous).
