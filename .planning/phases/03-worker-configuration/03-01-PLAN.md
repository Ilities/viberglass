<objective>
Define shared types for payload-based worker configuration

Purpose: Establish the contract between platform and workers for configuration delivery. Workers receive their complete configuration via the Lambda event/ECS task params/Docker env vars, including clanker metadata, credential variable names, and S3 URLs for large config files. This eliminates the need for workers to call the platform API.

Output: WorkerConfiguration, CredentialDescriptor, and ConfigFileUrl types in shared types package; extended CodingJobData with configuration fields
</objective>

  <files>packages/types/src/worker.ts</files>
  <action>
Create new file packages/types/src/worker.ts defining the worker configuration contract:

1. WorkerConfiguration interface - metadata passed in invocation payload:
   ```typescript
   export interface WorkerConfiguration {
     // Clanker identification
     clankerId: string;
     clankerName: string;
     clankerSlug: string;

     // Executor specification (how the worker should run)
     executorType: 'lambda' | 'ecs' | 'docker';
     executorConfig?: Record<string, unknown>;

     // Credential variable NAMES (not values)
     // Platform injects values as environment variables
     credentials: CredentialDescriptor[];

     // Large config files fetched from S3
     // Platform uploads these when clanker is created/updated
     configFiles?: ConfigFileUrl[];

     // Environment variables from clanker config
     // These are set before agent execution
     environmentVariables?: Record<string, string>;
   }
   ```

2. CredentialDescriptor interface - describes credential variables:
   ```typescript
   export interface CredentialDescriptor {
     // Environment variable name (e.g., GITHUB_TOKEN)
     name: string;
     // Credential type for validation
     type: 'token' | 'api_key' | 'basic' | 'ssh_key';
     // Which components need this credential
     requiredFor: ('scm' | 'api' | 'git' | 'agent')[];
     // Optional: which SCM provider if type='scm'
     provider?: 'github' | 'gitlab' | 'bitbucket' | 'generic';
   }
   ```

3. ConfigFileUrl interface - references large files in S3:
   ```typescript
   export interface ConfigFileUrl {
     // File type (agents.md, claude.md, etc.)
     fileType: string;
     // S3 URL or similar public/signed URL
     url: string;
     // Content type for fetch parsing
     contentType?: string;
   }
   ```

4. Export all types from packages/types/src/index.ts

Design notes:
- Credentials are NAMES only - platform injects values as env vars
- Large files (agents.md, claude.md) are referenced by URL, not embedded
- Workers fetch from S3 using platform's credentials, not tenant credentials
- This enables workers to run without calling the platform API
