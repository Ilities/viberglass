Create WorkerConfigurationClient class following CallbackClient pattern:

1. Import axios, Logger, and define ClankerConfig interface
2. Constructor takes (logger, config: {platformUrl, tenantId, cacheTtl?})
3. Implement getClankerConfig(clankerId: string) method:
   - Check in-memory cache first (Map<string, {data, expiry}>)
   - If cached and not expired, return cached data
   - Otherwise, fetch from GET {platformUrl}/api/clankers/{clankerId}
   - Headers: Content-Type: application/json, X-Tenant-Id: {tenantId}
   - Cache for 5 minutes (300000ms default)
   - Return response.data.data (the clanker object)

4. Define ClankerConfig interface matching platform response:
   - id, name, slug, description
   - deploymentStrategyId, deploymentConfig
   - environmentVariables: Record<string, string>
   - configFiles: {fileType, content}[]

5. Implement clearCache() method for testing

Use CallbackClient.ts as reference for axios patterns, error handling, and logging style. Do NOT implement retry logic in config fetch (unlike callback) - cache is sufficient for reliability.
