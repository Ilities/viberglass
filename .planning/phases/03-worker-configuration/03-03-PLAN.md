<task type="auto">
  <name>Update executeTask to use platform configuration</name>
  <files>viberator/app/src/workers/viberator.ts</files>
  <action>
Modify executeTask method to use platform-provided configuration:

1. At start of executeTask, after extracting data:
   - If data.clankerConfig exists, use it
   - Otherwise, fetch using getClankerConfig(data.clankerId or default)

2. Validate credentials BEFORE any work:
   - Call credentialInjector.validateCredentials(data.credentials || [])
   - This throws clear error if any required credential missing

3. Inject environment variables from clanker config:
   - Call credentialInjector.injectEnvironmentVariables(clankerConfig.environmentVariables || {})
   - This sets clanker-specified env vars before agent execution

4. Inject SCM credentials before git operations:
   - Before cloneRepositoryToWorkspace, extract SCM credential
   - Find credential with type matching repo provider (github/gitlab/bitbucket)
   - Read value: credentialInjector.getCredentialValue(cred.name)
   - Call SCMAuthFactory.setProviderToken(repository, token)

5. Clear credentials in finally block:
   - After job completion (success or failure)
   - Call SCMAuthFactory.clearDynamicCredentials()
   - This prevents Lambda container reuse from leaking credentials

6. Load agent instruction files from clanker config:
   - Check clankerConfig.configFiles for 'agents.md' or similar
   - If present, include in execution context for agent
   - Otherwise use existing behavior

Order: validate credentials -> inject env vars -> inject SCM -> execute agent -> clear credentials
  </action>

<task type="auto">
  <name>Update Lambda handler for credential cleanup</name>
  <files>viberator/app/src/workers/lambda-handler.ts</files>
  <action>
Update lambda-handler.ts to ensure credentials are cleared even if worker throws:

1. Add finally block to handler:
   ```typescript
   try {
     const result = await worker.executeTask(jobData);
     return { statusCode: 200, body: JSON.stringify(result) };
   } catch (error) {
     // existing error handling
   } finally {
     // CRITICAL: Clear injected credentials to prevent leakage
     SCMAuthFactory.clearDynamicCredentials();
   }
   ```

2. Pass platform configuration to worker constructor:
   - Read PLATFORM_API_URL from environment
   - Pass to ViberatorWorker constructor

3. Log credential cleanup for debugging

Lambda container reuse means process.env persists between invocations. Without cleanup, credentials from Job A could leak into Job B. The finally block ensures cleanup even on error.
  </action>
  <verify>grep -q "finally" viberator/app/src/workers/lambda-handler.ts && grep -q "clearDynamicCredentials" viberator/app/src/workers/lambda-handler.ts</verify>
  <done>Lambda handler clears credentials in finally block</done>
</task>
