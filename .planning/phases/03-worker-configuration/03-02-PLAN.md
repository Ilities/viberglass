  <name>Create ConfigLoader for fetching from S3 URLs</name>
  <files>viberator/app/src/workers/ConfigLoader.ts</files>
  <action>
Create ConfigLoader class for fetching config files from S3 URLs:

1. Constructor takes (logger: Logger)
2. loadConfigFiles(configFiles: ConfigFileUrl[]): Promise<Record<string, string>>
   - Fetches each file from its URL using fetch()
   - Uses fileType as the key in returned record
   - Supports S3 URLs (https://*.s3.*.amazonaws.com/*)
   - Optional 10-second timeout per file
   - Logs success/failure for each file
   - Throws if any file fails to load
3. loadSingleConfigFile(url: string): Promise<string>
   - Helper to fetch a single file
   - Returns text content
   - Handles content-type header for parsing
4. cache results in-memory Map<string, string> with URL as key
   - Avoids redundant fetches
   - clearCache() method for testing

Use native fetch (Node 18+) - no additional dependencies needed. Workers use platform's AWS credentials (assumed role), not tenant credentials.

Example usage:
```typescript
const loader = new ConfigLoader(logger);
const configs = await loader.loadConfigFiles(clankerConfig.configFiles);
// { 'agents.md': '...', 'claude.md': '...' }
```

<task type="auto">
  <name>Create CredentialInjector class</name>
  <files>viberator/app/src/workers/CredentialInjector.ts</files>
  <action>
Create CredentialInjector class to validate and inject credentials:

1. Constructor takes (logger: Logger)
2. validateCredentials(credentials: CredentialDescriptor[], requiredFor?: string[]): void
   - Checks each credential exists in process.env
   - If requiredFor provided, only check credentials where it matches
   - Throws Error with clear message if any missing: "Missing required credential: {name} (required for {requiredFor})"
   - Logs all found credentials (names only, not values) for debugging
3. injectEnvironmentVariables(envVars: Record<string, string>): void
   - Sets each key-value pair on process.env
   - Overwrites existing values if present
   - Logs what was injected (names only, not values per SEC-04)
4. clearInjectedCredentials(keys: string[]): void
   - Clears specified environment variables
   - Important for Lambda container reuse to prevent credential leakage
5. getCredentialValue(name: string): string | undefined
   - Helper to read credential from environment
   - Returns process.env[name]

Security note: Never log credential values, only variable names.
  </action>
