**Gathered:** 2026-01-23

## Phase Boundary

Remove the unused S3+CloudFront frontend static hosting infrastructure from the Pulumi codebase, since Amplify SSR is now the deployment method. The S3+CloudFront resources were defined in Phase 10 but never provisioned (pulumi up was never run), so this is pure code cleanup.

Additionally, bring Amplify infrastructure under Pulumi management (currently manual console setup) and update documentation.

## Implementation Decisions

### S3+CloudFront Cleanup
- Pulumi was never run for these resources — pure code deletion, no pulumi commands needed
- Removal order follows dependency chain: Origin Access Control → CloudFront Distribution → S3 Bucket
- Full deletion of code definitions (no commented-out references)
- Remove all SSM parameters for frontend CDN URL (`/viberator/*/frontend-cdn-url`)
- Files to clean up:
  - `infrastructure/frontend.ts` — S3 bucket, CloudFront distribution, OAC definitions
  - `infrastructure/index.ts` — frontend component registration
  - `infrastructure/README.md` — S3+CloudFront sections

### Amplify Pulumi Management
- Move Amplify from manual console setup to full Pulumi infrastructure-as-code
- Create Amplify app via Pulumi (AWS Amplify provider)
- Multi-branch single app pattern: one Amplify app with dev, staging, and prod branches
- Each branch gets its own environment variables and build settings
