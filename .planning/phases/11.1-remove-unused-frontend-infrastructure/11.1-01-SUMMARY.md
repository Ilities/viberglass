# Phase 11.1 Plan 01: Remove Unused Frontend Infrastructure Summary

Remove the unused S3+CloudFront frontend static hosting infrastructure from the Pulumi codebase, since Amplify SSR is now the deployment method.

## One-Liner

Deleted 250-line Pulumi component for S3+CloudFront frontend hosting that was never deployed, saving $15-20/month in AWS costs and simplifying infrastructure to align with actual Amplify SSR deployment method.

## What Was Done

### 1. Removed Frontend Component File
- **Deleted**: `infrastructure/components/frontend.ts` (250 lines)
- **Contained**: S3 bucket, CloudFront distribution, Origin Access Control, bucket policy, SSM parameters
- **Impact**: No AWS resource cleanup needed (never deployed with `pulumi up`)

### 2. Cleaned Pulumi Stack
- **Removed import**: `import { createFrontend, FrontendOutputs } from "./components/frontend"`
- **Removed invocation**: `createFrontend({ config, backendUrl, bucketPrefix, priceClass })`
- **Removed exports**:
  - `frontendUrl`
  - `frontendBucketName`
  - `frontendBucketArn`
  - `frontendCloudFrontDomain`
  - `frontendDistributionId`
  - `frontendSsmApiUrlPath`
  - `frontendSsmCdnUrlPath`
- **Result**: Clean Pulumi stack with only backend infrastructure

### 3. Updated Documentation
- **Removed from Overview**: S3+CloudFront static hosting reference
- **Updated Architecture diagram**: Removed CloudFront and FrontendS3 nodes
- **Updated User Request Flow**: Direct to ALB (no CloudFront intermediary)
- **Replaced deployment instructions**: S3+CloudFront sync commands → Amplify SSR setup
- **Added CI/CD reference**: `.github/workflows/deploy-frontend.yml`

### 4. Verified Compilation
- **Result**: TypeScript compilation succeeds without errors
- **Verification**: `tsc --noEmit` exits clean
- **Status**: Pulumi stack is type-safe and ready for deployment
