## Summary

This phase requires comprehensive test coverage for the worker execution flow implemented in Phase 4 and Phase 4.1. The research identified existing testing patterns in the codebase (credential system has 175 passing tests) and established best practices for mocking AWS SDK v3 clients (Lambda, ECS) and dockerode for Docker container operations.

**Primary recommendation:** Follow the established credential testing patterns for consistency. Use `jest.mock()` for AWS SDK v3 clients (same pattern as SSM in AwsSsmProvider.test.ts) and manual mocking for dockerode. For frontend, use React Testing Library with `jest-fetch-mock` or MSW for API mocking.

### Pattern 1: AWS SDK v3 Client Mocking (Established Pattern)
**What:** Mock AWS SDK clients using `jest.mock()` - same as AwsSsmProvider.test.ts
**When to use:** All Lambda and ECS invoker unit tests
**Example:**
```typescript
// Source: Existing pattern in AwsSsmProvider.test.ts
import { LambdaClient, InvokeCommand } from '@aws-sdk/client-lambda';

jest.mock('@aws-sdk/client-lambda', () => {
  return {
    LambdaClient: jest.fn().mockImplementation(() => ({
      send: jest.fn(),
    })),
    InvokeCommand: jest.fn().mockImplementation((input) => ({ input })),
  };
});

describe('LambdaInvoker', () => {
  let mockSend: jest.Mock;

  beforeEach(() => {
    jest.clearAllMocks();
    // Get mock from private instance
    const invoker = new LambdaInvoker();
    const mockClient = invoker['client'] as any;
    mockSend = mockClient.send;
  });

  it('should invoke Lambda function successfully', async () => {
    mockSend.mockResolvedValueOnce({
      $metadata: { requestId: 'test-request-id' },
      StatusCode: 202,
    });

    const result = await invoker.invoke(mockJob, mockClanker);

    expect(result.executionId).toBe('test-request-id');
    expect(result.workerType).toBe('lambda');
  });
});
```

### Anti-Patterns to Avoid
- **Hand-rolling AWS mock libraries:** Use jest.mock() pattern established in codebase
- **Testing implementation details:** Test behavior, not internal methods
- **Shared state between tests:** Always reset mocks in beforeEach
- **Not using TypeScript types:** Leverage existing types for mock data

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| AWS SDK mocking | Custom mock classes | jest.mock() pattern | Already established, consistent with codebase |
| Test fixtures | Manual setup in each test | Helper functions | Reuse patterns from credential tests |
| API response mocking | Custom response builders | jest.fn().mockResolvedValue() | Simple, built-in |
| Date mocking | Manual date injection | jest.useFakeTimers() | Built-in, handles time-dependent tests |
| Async retry testing | Manual wait loops | jest.advanceTimersByTime() | Test retry logic deterministically |

**Key insight:** The credential system already has 175 passing tests using these patterns. Follow them rather than inventing new ones.

## Common Pitfalls

### Pitfall 1: Forgetting to Reset Singleton Factories
**What goes wrong:** Tests fail due to leftover state from previous tests
**Why it happens:** WorkerInvokerFactory and other factories use singletons
**How to avoid:** Always call reset functions in beforeEach
```typescript
import { resetWorkerInvokerFactory } from '../../../workers/WorkerInvokerFactory';

beforeEach(() => {
  resetWorkerInvokerFactory();
});
```

### Pitfall 2: Mock Not Returning Expected Structure
**What goes wrong:** TypeError: Cannot read property of undefined
**Why it happens:** AWS SDK mocks don't return the full response structure
**How to avoid:** Always include `$metadata` in mock responses
```typescript
// Wrong
mockSend.mockResolvedValue({ StatusCode: 202 });

// Right
mockSend.mockResolvedValue({
  $metadata: { requestId: 'test-id' },
  StatusCode: 202,
});
```

### Pitfall 3: Async Tests Not Properly Awaited
**What goes wrong:** Tests pass when they should fail
**Why it happens:** Fire-and-forget async code (like worker invocation)
**How to avoid:** Use Promise.all() or mock all async dependencies
```typescript
// WorkerExecutionService fires worker invocation - test needs to handle this
await Promise.all([
  executeJobPromise,
  expect(mockInvoker.invoke).toHaveBeenCalled(),
]);
```

### Pitfall 4: Frontend Tests Using Real fetch
**What goes wrong:** Tests make real network calls
**Why it happens:** fetch not mocked before importing modules
**How to avoid:** Mock fetch before importing code under test
```typescript
// Before import
global.fetch = jest.fn();
// Then import
import { runTicket } from './job-api';
```

### Pitfall 5: Test Database Not Isolated
**What goes wrong:** Integration tests affect each other
**Why it happens:** Tests share database state
**How to avoid:** Use transactions with rollback or unique IDs
```typescript
// Use unique job IDs in each test
const jobId = `test-job-${Date.now()}-${Math.random()}`;
```

## Code Examples

### LambdaInvoker Test Pattern
```typescript
// Source: Based on AwsSsmProvider.test.ts pattern
describe('LambdaInvoker', () => {
  let invoker: LambdaInvoker;
  let mockSend: jest.Mock;
  let mockJob: JobData;
  let mockClanker: Clanker;

  beforeEach(() => {
    jest.clearAllMocks();
    invoker = new LambdaInvoker({ region: 'eu-west-1' });
    const mockClient = invoker['client'] as any;
    mockSend = mockClient.send;

    mockJob = {
      id: 'job-123',
      tenantId: 'tenant-1',
      repository: 'https://github.com/test/repo',
      task: 'Fix the bug',
      branch: null,
      baseBranch: 'main',
      context: {},
      settings: {},
      timestamp: Date.now(),
    };

    mockClanker = {
      id: 'clanker-1',
      deploymentStrategy: { name: 'lambda' },
      deploymentConfig: { functionName: 'test-function' },
    };
  });

  describe('invoke()', () => {
    it('should invoke Lambda with correct payload', async () => {
      mockSend.mockResolvedValueOnce({
        $metadata: { requestId: 'req-123' },
        StatusCode: 202,
      });

      const result = await invoker.invoke(mockJob, mockClanker);

      expect(result).toEqual({
        executionId: 'req-123',
        workerType: 'lambda',
      });
      expect(mockSend).toHaveBeenCalledTimes(1);
    });

    it('should throw permanent error when functionName missing', async () => {
      mockClanker.deploymentConfig = {};

      await expect(invoker.invoke(mockJob, mockClanker)).rejects.toThrow(
        'Lambda function name or ARN not configured'
      );
    });

    it('should classify TooManyRequestsException as transient', async () => {
      const error = new Error('TooManyRequestsException');
      (error as any).name = 'TooManyRequestsException';
      mockSend.mockRejectedValueOnce(error);

      const thrownError = await invoker.invoke(mockJob, mockClanker).catch(e => e);

      expect(thrownError.isTransient).toBe(true);
    });
  });
});
```

### WorkerExecutionService Retry Logic Test
```typescript
describe('WorkerExecutionService', () => {
  let service: WorkerExecutionService;
  let mockInvoker: jest.Mocked<WorkerInvoker>;

  beforeEach(() => {
    jest.clearAllMocks();
    jest.useFakeTimers();
    service = new WorkerExecutionService({ maxRetries: 3, baseDelayMs: 100 });
    mockInvoker = {
      name: 'MockInvoker',
      invoke: jest.fn(),
      isAvailable: jest.fn().mockResolvedValue(true),
    };
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('should retry on transient error', async () => {
    const transientError = new WorkerError(
      'Temporary failure',
      ErrorClassification.TRANSIENT
    );
    mockInvoker.invoke
      .mockRejectedValueOnce(transientError)
      .mockRejectedValueOnce(transientError)
      .mockResolvedValueOnce({
        executionId: 'exec-123',
        workerType: 'lambda',
      });

    const executePromise = service.executeJob(mockJob, mockClanker);

    // Fast-forward through retries
    await jest.advanceTimersByTimeAsync(100); // First retry delay
    await jest.advanceTimersByTimeAsync(200); // Second retry delay

    const result = await executePromise;

    expect(result.success).toBe(true);
    expect(result.attempts).toBe(3);
  });

  it('should fail immediately on permanent error', async () => {
    const permanentError = new WorkerError(
      'Config error',
      ErrorClassification.PERMANENT
    );
    mockInvoker.invoke.mockRejectedValue(permanentError);

    const result = await service.executeJob(mockJob, mockClanker);

    expect(result.success).toBe(false);
    expect(result.attempts).toBe(1);
  });
});
```
