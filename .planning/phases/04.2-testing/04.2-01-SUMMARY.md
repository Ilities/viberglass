## Issues Encountered

1. **Jest mock consumption issue**: Initial tests failed because multiple assertions on the same promise caused the mock to be consumed twice. Fixed by storing the promise in a variable and reusing it for assertions.

2. **Error object construction**: Simply passing plain objects to `mockRejectedValueOnce()` resulted in errors being wrapped as `TypeError`. Fixed by properly constructing Error objects with `name` and `$metadata` properties set.

3. **ECS conflict detection**: Discovered that Docker invoker requires BOTH "Conflict" AND "already in use" in the message for transient classification. Adjusted test to verify both positive and negative cases.
