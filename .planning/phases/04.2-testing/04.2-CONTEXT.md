# Phase 4.2: Testing - Context

**Gathered:** 2026-01-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Pragmatic testing for worker execution flow. Focus on error classification, retry behavior, and critical paths. Skip trivial tests. Scope covers the entire codebase, not just Phase 4/4.1 code.

</domain>

<decisions>
## Implementation Decisions

### Test scope
- Test the entire codebase, not just Phase 4/4.1
- Focus on critical paths: happy path + known failure modes
- Skip trivial code: simple getters, straightforward CRUD, thin wrappers
- Claude decides what qualifies as trivial

### Test layering
- **Unit tests**: Few, only for complex algorithms. Mock dependencies.
- **Integration tests**: Real testcontainers for database, mock AWS SDK and dockerode
- **E2E tests**: Testcontainers for DB, real dockerode, LocalStack for AWS

### Mock depth by layer
- Unit: Mock at dependency level
- Integration: Mock external services (AWS, Docker), real DB via testcontainers
- E2E: Real services where possible, LocalStack for AWS

### Database isolation
- Fresh database state per test suite (not per individual test)
- Tests within a suite can share state

### Test utilities package
- Create new dedicated test-utils package in monorepo
- Not just a folder — a proper package with its own dependencies

### Docker in E2E tests
- Real Docker if available, skip Docker tests if daemon not running
- Provides CI flexibility (not all runners have Docker)

### LocalStack for AWS
- E2E test suite starts LocalStack via docker-compose
- Required for E2E, not optional

### Integration vs E2E boundary
- Integration = multi-service with real DB + mocked external services
- E2E = full stack including real external services (Docker, LocalStack)

### Callback testing
- Test callback endpoint separately from worker invocation
- Don't require full worker->callback E2E flow in this phase

### Claude's Discretion
- Which specific code qualifies as "trivial" to skip
- Test file organization within the test-utils package
- Exact docker-compose configuration for LocalStack

</decisions>
