<task type="auto">
  <name>Task 1: Create RDS subnet group and parameter group</name>
  <files>infrastructure/components/database.ts</files>
  <action>
Create `infrastructure/components/database.ts` with RDS infrastructure:

1. Create DB Subnet Group:
   - Use private subnets from VPC component
   - Name: `${environment}-viberator-db-subnet-group`
   - Description: "Viberator database subnet group"

2. Create DB Parameter Group:
   - Family: postgres16 (or postgres15 if not available)
   - Parameters for performance tuning:
     - shared_buffers: { value: "{DBInstanceClassMemory/16384}", applyMethod: "immediate" }
     - max_connections: { value: "100", applyMethod: "pending-reboot" }
     - log_min_duration_statement: { value: "1000", applyMethod: "immediate" } - log slow queries
     - log_statement: { value: "all", applyMethod: "immediate" } - for debugging

3. Import VPC component to get privateSubnetIds and rdsSecurityGroupId

4. Export subnet group name and parameter group name

Use `aws.rds.SubnetGroup` and `aws.rds.ParameterGroup`.
  </action>
  <verify>
    - `cd infrastructure && npx tsc` compiles without errors
    - `pulumi preview --stack dev` shows DB subnet group and parameter group
    - Parameter group has PostgreSQL family set
  </verify>
  <done>
    DB subnet group references VPC private subnets. Parameter group configured for PostgreSQL performance. Both exported for use by RDS instance.
  </done>
</task>

<task type="auto">
  <name>Task 2: Generate and store database credentials in SSM</name>
  <files>infrastructure/components/database.ts</files>
  <action>
Add credential generation to `infrastructure/components/database.ts`:

1. Generate random database password:
   - Use `pulumi.random.RandomPassword` resource
   - Length: 32 characters
   - Include: special, upper, lower, numbers
   - Override special chars to avoid shell issues: "_%+-.=@:"

2. Create SSM Parameter for username:
   - Path: `/viberator/${environment}/database/username`
   - Type: String
   - Value: "viberator" (fixed username)

3. Create SSM Parameter for password:
   - Path: `/viberator/${environment}/database/password`
   - Type: SecureString
   - Value: From random password resource

4. Create SSM Parameter for connection string:
   - Path: `/viberator/${environment}/database/url`
   - Type: SecureString
   - Value: `postgresql://viberator:${password}@${endpoint}:5432/viberator`
   - Depends on RDS instance being created

5. Create SSM Parameter for host/endpoint:
   - Path: `/viberator/${environment}/database/host`
   - Type: String
   - Value: RDS endpoint

Note: The connection string SSM param will reference the RDS endpoint, creating a dependency.
  </action>
  <verify>
    - `pulumi preview --stack dev` shows RandomPassword resource
    - `pulumi preview --stack dev` shows 4 SSM parameters (username, password, url, host)
    - Password parameter uses SecureString type
  </verify>
  <done>
    Random password generated for database. All connection details stored in SSM. Password is SecureString. Connection string includes RDS endpoint.
  </done>
</task>

<task type="auto">
  <name>Task 3: Create RDS PostgreSQL instance</name>
  <files>infrastructure/components/database.ts</files>
  <action>
Add RDS instance to `infrastructure/components/database.ts`:

1. Create RDS Instance:
   - Engine: postgres
   - Engine version: 16.1 (or latest available)
   - Instance class: Configurable (default: db.t4g.micro for dev, db.t4g.large for staging, db.m6g.xlarge for prod)
   - Allocated storage: 20 GB (configurable)
   - Storage type: gp3 (cost-effective)
   - Storage encrypted: true
   - Database name: viberator
   - Master username: From SSM (or "viberator" directly)
   - Master password: From RandomPassword resource
   - VPC security group IDs: [rdsSecurityGroupId]
   - DB subnet group name: From subnet group resource
   - Parameter group name: From parameter group resource
   - Multi-AZ: false for dev/staging, true for prod (configurable)
   - Publicly accessible: false
   - Skip final snapshot: true for dev, false for prod
   - Deletion protection: false for dev, true for prod
   - Backup retention: 1 day for dev, 7 for prod
   - Monitoring interval: 0 for dev, 60 for prod (Enhanced Monitoring)

2. Add tags:
   - Environment: ${environment}
   - Application: viberator
   - Managed-by: pulumi

3. Export:
   - rdsEndpoint (instance.endpoint)
   - rdsPort (5432)
   - rdsInstanceArn
   - databaseName

Use `aws.rds.Instance` with configuration from stack config.
  </action>
  <verify>
    - `pulumi preview --stack dev` shows RDS instance in private subnets
    - Dev instance is db.t4g.micro, single-AZ, no deletion protection
    - Prod instance (when previewed) is db.m6g.xlarge, multi-AZ, deletion protection enabled
    - RDS security group only allows backend access
  </verify>
  <done>
    RDS PostgreSQL instance created in private subnets. Instance class scales with environment. Multi-AZ for production. Credentials stored securely. Connection available via SSM.
  </done>
</task>
