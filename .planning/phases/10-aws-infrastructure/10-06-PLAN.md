   - lambdaLogGroupArn

<action>
Update `infrastructure/components/worker-lambda.ts` to use logging component:

1. Import logging component
2. Update Lambda function configuration:
   - Remove default log group creation (let CloudWatch create it automatically)
   - Or explicitly set: `loggingConfig: { logGroup: lambdaLogGroupName }`
   - If using aws.lambda.Function, the log group is auto-created from the name

3. Ensure Lambda IAM role has CloudWatch Logs permissions:
   - logs:CreateLogGroup
   - logs:CreateLogStream
   - logs:PutLogEvents

   These are typically in AWSLambdaBasicExecutionRole managed policy.

4. Export log group name from Lambda component

The key is ensuring the Lambda uses the log group name from the logging component, so retention is applied.
  </action>

    - Log group name matches `/aws/lambda/viberator-${environment}-worker`
  </verify>

  </done>

<task type="auto">
  <name>Task 3: Update ECS task definitions to use log groups</name>
  <files>infrastructure/components/worker-ecs.ts, infrastructure/components/backend-ecs.ts</files>
<action>
Update ECS task definitions to use logging component:

1. For worker ECS task definition:
   - Import logging component
   - Update containerDefinitions logConfiguration:
   ```json
   "logConfiguration": {
     "logDriver": "awslogs",
     "options": {
       "awslogs-group": ecsWorkerLogGroupName,
       "awslogs-region": aws.config.region,
       "awslogs-stream-prefix": "worker"
     }
   }
   ```

2. For backend ECS task definition (to be created in plan 10-07):
   - Same pattern but use backendLogGroupName
   - Stream prefix: "backend"

3. Ensure ECS task execution role has CloudWatch Logs permissions (part of AmazonECSTaskExecutionRolePolicy)

4. Export log group names from ECS components

The log group must be created with retention BEFORE the task definition references it.
  </action>
  <verify>
    - `pulumi preview --stack dev` shows ECS task with awslogs driver
    - Log group matches `/ecs/viberator-${environment}-worker`
  </verify>
  <done>
    ECS tasks write to CloudWatch with proper log driver. Log retention applies. Both worker and backend use correct log groups.
  </done>
</task>
