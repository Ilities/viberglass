<task type="auto">
  <name>Task 1: Create KMS key with key policy</name>
  <files>infrastructure/components/kms.ts</files>
  <action>
Create `infrastructure/components/kms.ts` with KMS infrastructure:

1. Create KMS Key:
   - Use `aws.kms.Key`
   - Description: "Viberator SSM encryption key for ${environment}"
   - Enable key rotation: true
   - Tags: Environment, Application, Managed-by

2. Create Key Policy:
   - Default policy allows root account full access
   - Add statements for Lambda/ECS roles to use the key
   - Key administrators (for key management): AWS account root
   - Key users (for encryption/decryption): Lambda role, ECS task role, backend role

3. Key policy structure:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "EnableRoot",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::${accountId}:root"},
      "Action": "kms:*",
      "Resource": "*"
    },
    {
      "Sid": "AllowRolesUse",
      "Effect": "Allow",
      "Principal": {"AWS": ["${lambdaRoleArn}", "${ecsTaskRoleArn}", "${backendRoleArn}"]},
      "Action": ["kms:Decrypt", "kms:GenerateDataKey*"],
      "Resource": "*"
    }
  ]
}
```

4. Create KMS Alias:
   - Use `aws.kms.Alias`
   - Name: alias/viberator-${environment}-ssm
   - Target key ID: from KMS key

5. Export key ID and ARN

Note: Since roles are created in other components, use `pulumi.interpolate` or reference role ARNs from those components.
  </action>
  <verify>
    - `cd infrastructure && npx tsc` compiles without errors
    - `pulumi preview --stack dev` shows KMS key with rotation enabled
    - Key policy allows decrypt for Lambda/ECS roles
  </verify>
  <done>
    KMS key created with automatic rotation. Key policy allows compute roles to decrypt. Alias created for easy reference.
  </done>
</task>
