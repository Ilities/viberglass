# 18-10 Plan: Jira Targeted Configuration UI

**Phase:** 18-targeted-webhook-integration-rollout  
**Plan:** 10  
**PR:** PR-10  
**Estimate:** 1 day  
**Status:** Planned  
**Depends on:** 18-04, 18-08, 18-09

## Objective

Deliver Jira-specific webhook configuration UI with explicit inbound/outbound Jira semantics.

## Scope

In scope:
- Jira inbound config section
- Jira outbound config section
- Jira setup instructions and validation hints

Out of scope:
- non-Jira provider UI

## UI Behavior

Inbound section should expose:
- Jira inbound endpoint URL
- secret handling controls
- event allowlist controls mapped to Jira terms
- auto-execute toggle

Outbound section should expose:
- lifecycle event toggles
- token placeholder/rotation guidance
- Jira project/issue mapping preview

## Implementation Tasks

1. Add Jira-specific section components and wire to integration detail page.
2. Replace generic delivery and payload help text with Jira-focused copy.
3. Ensure event toggles align with supported Jira outbound/inbound behavior.
4. Validate required Jira fields before save actions.

## Candidate Files

- `apps/platform-frontend/src/pages/settings/integration-detail/*`
- `apps/platform-frontend/src/pages/settings/IntegrationDetailPage.tsx`
- `apps/platform-frontend/src/service/api/integration-api.ts`

## Test Plan

1. Component tests for Jira-specific controls and validation.
2. E2E for saving Jira webhook settings and reloading persisted state.
3. E2E for inbound config selection where multiple configs exist.

## Risks and Mitigations

- Risk: user confusion due to Jira-specific terminology.  
  Mitigation: explicit labels, examples, and endpoint setup instructions.

## Acceptance Criteria

1. Jira settings page is targeted and no longer generic.
2. Saved values map directly to Jira inbound/outbound backend contracts.
3. UI supports multiple inbound Jira webhook configs cleanly.
4. E2E verifies end-to-end save/load behavior.
