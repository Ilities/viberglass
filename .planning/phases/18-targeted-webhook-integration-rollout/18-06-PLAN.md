# 18-06 Plan: GitHub Outbound

**Phase:** 18-targeted-webhook-integration-rollout  
**Plan:** 06  
**PR:** PR-06  
**Estimate:** 1 day  
**Status:** Planned  
**Depends on:** 18-05

## Objective

Complete GitHub outbound publishing for job lifecycle feedback, using provider-targeted formatting and API calls.

## Scope

In scope:
- outbound event handling for GitHub configured outbound webhooks
- comment and label update behavior on job start and completion/failure
- outbound target resolution by ticket/inbound config/integration linkage

Out of scope:
- targeted GitHub UI changes (18-07)

## Provider Behavior

1. For `job_started`, post a concise kickoff comment.
2. For `job_ended`, post result summary and update labels.
3. Respect event toggles in outbound config.
4. Fail safely without breaking job status progression.

## Implementation Tasks

1. Harden outbound target resolution in `FeedbackService` for GitHub-specific cases.
2. Ensure provider token and repository/project resolution are explicit.
3. Improve retry and logging for transient GitHub API failures.
4. Validate label update behavior against current issue labels.

## Candidate Files

- `apps/platform-backend/src/webhooks/FeedbackService.ts`
- `apps/platform-backend/src/webhooks/providers/github-provider.ts`
- `apps/platform-backend/src/services/JobService.ts`

## Test Plan

1. Unit tests for outbound start/end event dispatch.
2. Tests for disabled events and missing external ticket IDs.
3. Tests for provider API error handling and non-blocking behavior.

## Risks and Mitigations

- Risk: outbound failures appearing as job failures.  
  Mitigation: outbound is asynchronous and logs non-blocking failures.
- Risk: repository mismatch between inbound and outbound config.  
  Mitigation: explicit resolution precedence and test fixtures.

## Acceptance Criteria

1. GitHub outbound posts correct comments and label updates.
2. Event toggles in outbound config are honored.
3. Outbound failures are recorded but do not break job completion state.
4. Unit tests cover dispatch and failure handling.
