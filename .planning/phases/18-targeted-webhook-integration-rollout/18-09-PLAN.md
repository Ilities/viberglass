# 18-09 Plan: Jira Outbound

**Phase:** 18-targeted-webhook-integration-rollout  
**Plan:** 09  
**PR:** PR-09  
**Estimate:** 1 day  
**Status:** Planned  
**Depends on:** 18-08

## Objective

Implement Jira-targeted outbound result publishing for job lifecycle updates.

## Scope

In scope:
- Jira comment publishing for job started/ended events
- Jira label/status updates where configured
- Jira-specific formatting and API path handling

Out of scope:
- Jira UI controls (18-10)

## Provider Behavior

1. Publish job-start message to Jira issue.
2. Publish completion/failure summary with PR link if present.
3. Apply status/label updates based on configured behavior.
4. Respect outbound event toggles.

## Implementation Tasks

1. Harden Jira provider outbound HTTP client configuration and endpoint paths.
2. Ensure issue key resolution from ticket metadata is reliable.
3. Add clear failure logging and retry-safe behavior in outbound dispatch.
4. Ensure outbound dispatch remains non-blocking to job status updates.

## Candidate Files

- `apps/platform-backend/src/webhooks/providers/jira-provider.ts`
- `apps/platform-backend/src/webhooks/FeedbackService.ts`
- `apps/platform-backend/src/services/JobService.ts`

## Test Plan

1. Provider unit tests for comment, label, and status update operations.
2. FeedbackService tests for Jira target resolution and event toggles.
3. Failure-path tests for API errors and missing issue key.

## Risks and Mitigations

- Risk: Jira API path/tenant config mismatch.  
  Mitigation: explicit `instanceUrl` usage and contract validation in config checks.
- Risk: formatted message rendering issues in Jira editor format.  
  Mitigation: normalize output and test markdown/ADF conversion.

## Acceptance Criteria

1. Jira outbound posts job lifecycle updates to the correct issue.
2. Optional label/status updates execute as configured.
3. Outbound failures do not impact final job state transitions.
4. Tests cover successful and failure scenarios.
