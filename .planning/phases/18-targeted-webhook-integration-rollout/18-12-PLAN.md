# 18-12 Plan: Shortcut Targeted Configuration UI

**Phase:** 18-targeted-webhook-integration-rollout  
**Plan:** 12  
**PR:** PR-12  
**Estimate:** 1 day  
**Status:** Planned  
**Depends on:** 18-04, 18-11

## Objective

Create a Shortcut-targeted webhook configuration page with dedicated inbound/outbound controls.

## Scope

In scope:
- Shortcut-specific inbound section
- Shortcut-specific outbound section
- setup instructions and event guidance

Out of scope:
- custom multi-webhook UI

## UI Behavior

Inbound section should expose:
- Shortcut endpoint URL and signature secret controls
- story/comment event toggles
- auto-execute toggle
- delivery history by selected inbound config

Outbound section should expose:
- outbound lifecycle event toggles
- token controls
- story targeting hints

## Implementation Tasks

1. Add Shortcut-specific section components.
2. Wire provider-specific field labels and help text.
3. Ensure all save/delete/retry actions are config-scoped.
4. Add concise setup copy for Shortcut webhook configuration in external UI.

## Candidate Files

- `apps/platform-frontend/src/pages/settings/integration-detail/*`
- `apps/platform-frontend/src/pages/settings/IntegrationDetailPage.tsx`

## Test Plan

1. Component tests for Shortcut inbound/outbound controls.
2. E2E: configure Shortcut integration, save settings, reload and verify.
3. E2E: delivery refresh and retry action paths.

## Risks and Mitigations

- Risk: feature parity drift across provider pages.  
  Mitigation: define shared UX baseline and provider-specific deltas explicitly.

## Acceptance Criteria

1. Shortcut integration detail is provider-targeted and non-generic.
2. Inbound/outbound settings map to Shortcut backend contract fields.
3. Delivery and retry interactions are config-scoped and functional.
4. E2E verifies essential flows.
