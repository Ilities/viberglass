# 18-07 Plan: GitHub Targeted Configuration UI

**Phase:** 18-targeted-webhook-integration-rollout  
**Plan:** 07  
**PR:** PR-07  
**Estimate:** 1 day  
**Status:** Planned  
**Depends on:** 18-04, 18-05, 18-06

## Objective

Replace generic webhook controls with GitHub-targeted inbound/outbound configuration UI.

## Scope

In scope:
- GitHub-specific inbound section
- GitHub-specific outbound section
- clear setup guidance and event-specific options

Out of scope:
- Jira/Shortcut/Custom UI

## UI Behavior

Inbound section should expose:
- GitHub webhook URL
- webhook secret regenerate/show/copy controls
- allowed inbound events
- auto-execute control
- delivery history scoped to selected inbound config

Outbound section should expose:
- outbound event toggles (`job_started`, `job_ended`)
- API token management
- repository/project mapping preview

## Implementation Tasks

1. Create GitHub-specific section components and wire into integration detail page.
2. Remove generic phrasing for GitHub and replace with GitHub setup instructions.
3. Support config selection if multiple inbound GitHub configs exist.
4. Ensure all calls use integration entity id and config id.

## Candidate Files

- `apps/platform-frontend/src/pages/settings/IntegrationDetailPage.tsx`
- `apps/platform-frontend/src/pages/settings/integration-detail/*`
- `apps/platform-frontend/src/service/api/integration-api.ts`

## Test Plan

1. UI component tests for inbound/outbound sections.
2. E2E scenario: configure GitHub webhook, save, and verify persisted values.
3. E2E scenario: multiple inbound configs selectable.

## Risks and Mitigations

- Risk: UI complexity increases with provider-specific sections.  
  Mitigation: isolate section components per provider and share only low-level controls.

## Acceptance Criteria

1. GitHub configuration page no longer uses generic webhook copy.
2. Inbound and outbound controls map to GitHub-specific behavior.
3. Delivery list reflects selected GitHub inbound config.
4. Existing integration detail behavior remains stable for non-GitHub providers.
