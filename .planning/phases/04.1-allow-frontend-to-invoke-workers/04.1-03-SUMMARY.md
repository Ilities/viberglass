## RunTicketModal Component

The modal provides pre-flight confirmation before executing a job:

**Features:**
- Displays ticket title and description (read-only)
- Dropdown to select from active clankers (with deployment config)
- Button shows selected clanker name (not generic "Run")
- Loading state ("Starting...") during API call
- Toast notification with "View Job" action link
- Immediate navigation to job detail page on success
- Warning message and "Configure Clankers" link when no active clankers

**Props:**
```typescript
interface RunTicketModalProps {
  ticket: Ticket | null
  clankers: Clanker[]
  project: string
  open: boolean
  onClose: () => void
}
```

**Clanker filtering:** Only shows clankers where `status === 'active'` AND `deploymentStrategyId` is truthy.

## TicketsTable Component

Converted the tickets page from server-side table to client component with interactive run buttons:

**Features:**
- Play icon button in each row's Actions column
- Click prevention for row navigation when clicking play button
- Tooltip indicates availability ("Run with clanker" or "No active clankers")
- Integrated RunTicketModal for clanker selection and execution
- Empty state when no tickets match filters

**Props:**
```typescript
interface TicketsTableProps {
  tickets: TicketSummary[]
  fullTickets: Ticket[]
  clankers: Clanker[]
  project: string
}
```

**Page changes:**
- Server component fetches clankers and full tickets
- Passes data to client component
- Filters remain server-side

## TicketRunButton Component

Added branded run button to ticket detail page:

**Features:**
- Shows default clanker name (e.g., "Run with MyClanker")
- Falls back to "Run" when no active clankers
- Opens RunTicketModal on click
- Positioned first in actions section (before "Enhance & Fix")

**Props:**
```typescript
interface TicketRunButtonProps {
  ticket: Ticket
  clankers: Clanker[]
  project: string
}
```

## User Flow

1. **From ticket list:**
   - User sees play icon button in each ticket row
   - Clicking play opens RunTicketModal
   - Modal shows ticket details and clanker dropdown
   - User confirms, sees "Starting..." on button
   - Toast appears with "View Job" link
   - Page navigates to job detail page

2. **From ticket detail:**
   - User sees "Run with [ClankerName]" button in actions
   - Clicking opens same RunTicketModal
   - Same flow as list view

3. **No clankers available:**
   - Button still visible but shows warning
   - Modal displays amber warning box
   - "Configure Clankers" button links to /clankers

## Error Handling

- API errors trigger toast with error message
- Loading state prevents double-submission
- `setIsRunning(false)` only on error (success case navigates away)

## Integration Points

- **job-api.ts**: Uses `runTicket(ticketId, clankerId)` from 04.1-02
- **sonner**: Toast library (installed in 04.1-02) for notifications
- **Next.js router**: Navigation to job detail page
- **@viberator/types**: Shared Ticket and Clanker types
