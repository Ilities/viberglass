## API Endpoint

### POST /api/tickets/:id/run

**Request body:**
```json
{
  "clankerId": "uuid"  // Required - clanker to use for execution
}
```

**Response (202 Accepted):**
```json
{
  "success": true,
  "data": {
    "jobId": "job_xxx",
    "status": "active"
  }
}
```

**Validation flow:**
1. Validates ticketId is valid UUID format
2. Validates clankerId is present and valid UUID
3. Returns 404 if ticket not found
4. Returns 500 if project missing (data integrity)
5. Returns 400 if project has no repositoryUrl
6. Returns 404 if clanker not found
7. Returns 400 if clanker has no deploymentStrategyId
8. Returns 400 if clanker status is not 'active'

**Fire-and-forget pattern:**
- Worker invocation is non-blocking
- Response returned immediately after job creation
- Worker errors logged but don't affect HTTP response
- Worker runs asynchronously via WorkerExecutionService

## JobService Changes

Added `SubmitJobOptions` interface to support optional ticket and clanker references:

```typescript
export interface SubmitJobOptions {
  ticketId?: string;
  clankerId?: string;
}
```

The `submitJob()` method now accepts an optional second parameter to store these references when creating jobs from tickets.
