    4. Sets up Node.js (actions/setup-node@v4) with node-version-file: package.json

    6. Installs dependencies: npm ci --legacy-peer-deps
    7. Runs tests: npm run test:unit -w @viberator/platform-backend
    8. Runs TypeScript compilation check: npm run build -w @viberator/platform-backend

    Use workspace-specific commands to avoid building all packages.
  </action>

<task type="auto">
  <name>Task 2: Create dev deployment workflow (auto-deploy on main)</name>
  <files>.github/workflows/deploy-backend-dev.yml</files>
  <action>
    Create .github/workflows/deploy-backend-dev.yml that:
    1. Triggers on push to main branch with paths: platform/backend/**, packages/types/**
    2. Runs on ubuntu-latest with permissions: id-token: write, contents: read
    3. Checks out code
    4. Configures AWS credentials via OIDC:
       - uses: aws-actions/configure-aws-credentials@v4
       - role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
       - aws-region: us-east-1
    5. Logins to ECR: aws-actions/amazon-ecr-login@v2
    6. Builds and pushes Docker image:
       - uses: docker/build-push-action@v6
       - context: .
       - file: platform/backend/Dockerfile.prod
       - push: true
       - tags: |
           ${{ steps.login-ecr.outputs.registry }}/viberator-backend:${{ github.sha }}
           ${{ steps.login-ecr.outputs.registry }}/viberator-backend:latest
       - cache-from: type=gha
       - cache-to: type=gha,mode=max
    7. Gets DATABASE_URL from SSM Parameter Store
    8. Runs migrations:
       - aws ssm get-parameter --name /viberator/dev/database/url --with-decryption
       - npm run migrate:latest -w @viberator/platform-backend
    9. Updates ECS service:
       - uses: aws-actions/amazon-ecs-deploy-task-definition@v1
       - service: dev-viberator-backend
       - cluster: dev-viberator-cluster
       - wait-for-service-stability: true

    Environment-specific values: dev
  </action>
  <verify>
    cat .github/workflows/deploy-backend-dev.yml
    Verify workflow has all required steps in correct order
  </verify>
  <done>
  Dev deployment workflow triggers on main branch push and deploys to dev environment
  </done>
</task>
