<task type="auto">
  <name>Task 1: Create Pulumi preview workflow for PR validation</name>
  <files>.github/workflows/pulumi-preview.yml</files>
  <action>
    Create .github/workflows/pulumi-preview.yml that:
    1. Triggers on pull_request with paths: infrastructure/**
    2. Runs on ubuntu-latest with permissions: id-token: write, contents: read, pull-requests: write
    3. Checks out code (actions/checkout@v4)
    4. Sets up Node.js (actions/setup-node@v4) with node-version-file: package.json
    5. Configures AWS credentials via OIDC:
       - uses: aws-actions/configure-aws-credentials@v4
       - role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
       - aws-region: eu-west-1
    6. Installs dependencies in infrastructure/:
       - working-directory: infrastructure
       - run: npm install
    7. Runs Pulumi preview with PR comment:
       - uses: pulumi/actions@v6
       - command: preview
       - stack-name: dev
       - work-dir: infrastructure
       - comment-on-pr: true
    8. Sets PULUMI_ACCESS_TOKEN from secrets

    Use dev stack for preview as the baseline comparison.
    Do NOT create staging workflow - only dev and prod per project decisions.
  </action>
  <verify>
    cat .github/workflows/pulumi-preview.yml
    Verify permissions include pull-requests: write for commenting
  </verify>
  <done>
    Pulumi preview runs on infrastructure PRs and posts summary as comment
