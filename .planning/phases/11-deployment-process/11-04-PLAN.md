<task type="auto">
  <name>Task 2: Create dev deployment workflow (auto-deploy on main)</name>
  <files>.github/workflows/deploy-frontend-dev.yml</files>
  <action>
    Create .github/workflows/deploy-frontend-dev.yml that:
    1. Triggers on push to main branch with paths: platform/frontend/**, packages/types/**
    2. Runs on ubuntu-latest with permissions: id-token: write, contents: read
    3. Checks out code
    4. Sets up Node.js with npm cache
    5. Installs dependencies: npm ci --legacy-peer-deps
    6. Configures AWS credentials via OIDC:
       - uses: aws-actions/configure-aws-credentials@v4
       - role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
       - aws-region: us-east-1
    7. Builds frontend with environment variables:
       - NEXT_PUBLIC_API_URL: ${{ secrets.DEV_API_URL }}
       - npm run build -w @viberator/frontend
    8. Syncs to S3:
       - aws s3 sync platform/frontend/out s3://${{ secrets.DEV_FRONTEND_BUCKET }} --delete
       - _next/static gets cache-control: "public, max-age=31536000, immutable"
       - Other files get cache-control: "public, max-age=86400"
    9. Invalidates CloudFront:
       - aws cloudfront create-invalidation --distribution-id ${{ secrets.DEV_CLOUDFRONT_ID }} --paths "/*"

    For smart invalidation (better than /*):
       - "/index.html"
       - "/_next/*"

    Environment-specific: dev bucket, dev CloudFront ID
  </action>
  <verify>
    cat .github/workflows/deploy-frontend-dev.yml
  </verify>
  <done>
  Dev frontend deployment triggers on main push and deploys to S3+CloudFront
  </done>
</task>
