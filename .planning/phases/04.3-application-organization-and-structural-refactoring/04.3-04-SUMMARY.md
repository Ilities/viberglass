## Decisions Made

1. **Created AgentCLIResult as separate type** - Rather than modifying ExecutionResult to have optional cost (which would break existing code), created an intermediate type that extends Omit<ExecutionResult, "executionTime" | "cost"> with optional cost. This keeps ExecutionResult pure while allowing flexibility.

2. **Exported JobStatus from JobService** - Instead of duplicating the status type definition across files, exported it from JobService as a reusable type. This ensures consistency if the database schema changes.

3. **Used indexed access types for literal casts** - For environment variable casts like `process.env.LOG_LEVEL as Configuration["logging"]["level"]`, used TypeScript's indexed access types. This provides type safety without needing to define duplicate literal types.
