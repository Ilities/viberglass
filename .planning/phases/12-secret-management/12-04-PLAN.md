       ```yaml

       ```

    3. Update SSM parameter fetch for backend URL to use standardized path:
       ```yaml
       - name: Get deployment config from SSM
         id: get-config
         run: |
           API_URL=$(aws ssm get-parameter \
             --name "/viberator/dev/frontend/apiUrl" \
             --query Parameter.Value \
             --output text)

           echo "backendUrl=$API_URL" >> $GITHUB_OUTPUT
           echo "NEXT_PUBLIC_API_URL=$API_URL" >> $GITHUB_ENV
       ```

    4. Add region configuration from SSM:
       ```yaml
       AWS_REGION=$(aws ssm get-parameter \
         --name "/viberator/dev/deployment/region" \
         --query Parameter.Value \
         --output text 2>/dev/null || echo "us-east-1")
       ```

    5. Update Configure AWS credentials step to use ${{ env.AWS_REGION }}

    6. Keep existing Amplify deployment wait loop

    7. Remove hardcoded fallback values like "https://dev-api.viberator.internal"
