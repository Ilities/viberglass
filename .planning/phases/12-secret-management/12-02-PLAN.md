<task type="auto">
  <name>Create Pulumi secrets component</name>
  <files>infrastructure/components/secrets.ts</files>
  <action>
    Create infrastructure/components/secrets.ts with:

    1. Define DeploymentSecretsOptions interface:
       - config: InfrastructureConfig (environment, region, tags, kmsKeyId)
       - databaseUrl: pulumi.Input<string>
       - databaseHost: pulumi.Input<string>
       - frontendApiUrl: pulumi.Input<string>
       - amplifyAppId?: pulumi.Input<string>
       - ecrRepository?: pulumi.Input<string>
       - ecsCluster?: pulumi.Input<string>
       - oidcRoleArn?: pulumi.Input<string>

    2. Define DeploymentSecretsOutputs interface:
       - All SSM parameter ARNs for dependency references
       - urlPath, hostPath, apiUrlPath, appIdPath, etc.

    3. Create createDeploymentSecrets(options: DeploymentSecretsOptions) function:

    For each secret, create aws.ssm.Parameter with:
    - Name: /viberator/{environment}/{category}/{key}
    - Type: "SecureString" for sensitive values, "String" for non-sensitive
    - Value: from options
    - KmsKeyId: from config.kmsKeyId for SecureString
    - Tags: from config.tags

    Secret categories:
    a) database (SecureString):
       - /viberator/{env}/database/url
       - /viberator/{env}/database/host

    b) frontend (String - not sensitive):
       - /viberator/{env}/frontend/apiUrl

    c) amplify (String - not sensitive):
       - /viberator/{env}/amplify/appId
       - /viberator/{env}/amplify/branch

    d) ecs (String - not sensitive):
       - /viberator/{env}/ecs/cluster
       - /viberator/{env}/ecs/service

    e) deployment (SecureString - OIDC role is sensitive):
       - /viberator/{env}/deployment/oidcRoleArn
       - /viberator/{env}/deployment/ecrRepository

    Return outputs with ARNs and names for workflow references.
  </action>
  <verify>TypeScript compiles in infrastructure/: npx tsc --noEmit</verify>
  <done>Pulumi secrets component creates SSM parameters with proper types and categorization</done>
</task>
