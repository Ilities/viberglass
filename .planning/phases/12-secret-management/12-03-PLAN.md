  <name>Update dev backend deployment workflow</name>
  <files>.github/workflows/deploy-backend-dev.yml</files>
  <action>
    Update .github/workflows/deploy-backend-dev.yml:

    1. Add environment directive to job:
       ```yaml
       jobs:
         deploy:
           environment: dev
       ```

    2. Replace hardcoded env values with environment secrets:
       - Remove: AWS_REGION: eu-west-1 (hardcoded)
       - Remove: ECR_REPOSITORY: viberator-backend (hardcoded)
       - Remove: ECS_CLUSTER: dev-viberator-cluster (hardcoded)
       - Remove: ECS_SERVICE: dev-viberator-backend (hardcoded)

    3. Add step to fetch deployment configuration from SSM:
       ```yaml
       - name: Get deployment config from SSM
         id: get-config
         run: |
           # Fetch deployment configuration
           AWS_REGION=$(aws ssm get-parameter --name "/viberator/dev/deployment/region" --query Parameter.Value --output text 2>/dev/null || echo "eu-west-1")
           ECR_REPOSITORY=$(aws ssm get-parameter --name "/viberator/dev/deployment/ecrRepository" --query Parameter.Value --output text 2>/dev/null || echo "viberator-backend")
           ECS_CLUSTER=$(aws ssm get-parameter --name "/viberator/dev/ecs/cluster" --query Parameter.Value --output text 2>/dev/null || echo "dev-viberator-cluster")
           ECS_SERVICE=$(aws ssm get-parameter --name "/viberator/dev/ecs/service" --query Parameter.Value --output text 2>/dev/null || echo "dev-viberator-backend")

           echo "aws_region=$AWS_REGION" >> $GITHUB_OUTPUT
           echo "ecr_repository=$ECR_REPOSITORY" >> $GITHUB_OUTPUT
           echo "ecs_cluster=$ECS_CLUSTER" >> $GITHUB_OUTPUT
           echo "ecs_service=$ECS_SERVICE" >> $GITHUB_OUTPUT
           echo "AWS_REGION=$AWS_REGION" >> $GITHUB_ENV
           echo "ECR_REPOSITORY=$ECR_REPOSITORY" >> $GITHUB_ENV
           echo "ECS_CLUSTER=$ECS_CLUSTER" >> $GITHUB_ENV
           echo "ECS_SERVICE=$ECS_SERVICE" >> $GITHUB_ENV
       ```

    4. Update AWS credentials step to use environment secret:
       ```yaml
       - name: Configure AWS credentials via OIDC
         uses: aws-actions/configure-aws-credentials@v4
         with:
           role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
           aws-region: ${{ env.AWS_REGION }}
       ```

    5. Keep existing DATABASE_URL fetch step (already uses SSM)

    6. Update references to use ${{ env.ECR_REPOSITORY }}, ${{ env.ECS_CLUSTER }}, ${{ env.ECS_SERVICE }}
  </action>
  <verify>Workflow YAML is valid: yamllint .github/workflows/deploy-backend-dev.yml</verify>
