<task type="auto">
  <name>Task 1: Create docker-compose.yml at repository root</name>
  <files>docker-compose.yml</files>
  <action>
    Create docker-compose.yml at repository root with the following services:

    **Version:** Use '3.8' for compatibility

    **Networks:** Create named network "viberator-dev-network"

    **Volumes:** Create "postgres-data" named volume

    **Service: postgres**
    - Image: postgres:16-alpine
    - Environment: POSTGRES_DB=viberator, POSTGRES_USER=viberator, POSTGRES_PASSWORD=viberator_dev
    - Ports: "5432:5432"
    - Healthcheck: pg_isready -U viberator, interval 5s, timeout 5s, retries 5
    - Volume: postgres-data:/var/lib/postgresql/data

    **Service: redis**
    - Image: redis:7-alpine
    - Ports: "6379:6379"
    - Healthcheck: redis-cli ping, interval 5s, timeout 5s, retries 5

    **Service: backend**
    - Build: context=./platform/backend, dockerfile=Dockerfile.dev
    - Environment:
      - DB_HOST=postgres
      - DB_PORT=5432
      - DB_NAME=viberator
      - DB_USER=viberator
      - DB_PASSWORD=viberator_dev
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - NODE_ENV=development
      - CHOKIDAR_USEPOLLING=true (for hot-reload in containers)
    - Ports: "8888:8888"
    - Volumes:
      - ./platform/backend:/app:cached
      - /app/node_modules (anonymous volume to preserve container's node_modules)
    - Depends on:
      - postgres: condition: service_healthy
      - redis: condition: service_healthy

    **Service: frontend**
    - Build: context=./platform/frontend, dockerfile=Dockerfile.dev
    - Environment:
      - NEXT_PUBLIC_API_URL=http://localhost:8888
      - NODE_ENV=development
      - CHOKIDAR_USEPOLLING=true
    - Ports: "3000:3000"
    - Volumes:
      - ./platform/frontend:/app:cached
      - /app/node_modules
      - /app/.next (preserve Next.js build cache)
    - Depends on:
      - backend (service_started, not healthchecked - backend ready enough)

    Use the e2e-tests/docker/docker-compose.e2e.yaml as reference for healthcheck patterns.
    Follow patterns from RESEARCH.md for cross-platform networking and volume mounts.

    DO NOT include worker service in this compose - workers are ephemeral and invoked separately via DockerInvoker.
  </action>
  <verify>docker compose config > /dev/null 2>&1 && echo "Compose config valid" || echo "Compose config invalid"</verify>
  <done>
    docker-compose.yml exists at repository root, passes validation with `docker compose config`, defines postgres/redis/backend/frontend services with proper healthchecks and dependency ordering
  </done>
</task>
