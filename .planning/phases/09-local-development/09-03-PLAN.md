  <action>
    Create docs/LOCAL_DEVELOPMENT.md with simplified one-command setup:

    **Title:** Local Development with Docker Compose

    **Overview:** One-command setup for complete Viberator platform locally

    **Prerequisites section:**
    - Docker Desktop or Docker Engine 20.10+
    - docker compose v2 (built into Docker Desktop)
    - No need to install Node.js, PostgreSQL, Redis separately (all in containers)

    **Quick Start section:**
    ```bash
    # Clone and navigate to repository
    cd viberator

    # Start all services
    docker compose up

    # Or detached mode
    docker compose up -d

    # Frontend: http://localhost:3000
    # Backend API: http://localhost:8888
    # PostgreSQL: localhost:5432
    # Redis: localhost:6379
    ```

    **First-time setup section:**
    - Run database migrations: `docker compose exec backend npm run migrate:latest`
    - Build worker image (for local Docker clankers): reference LOCAL_DOCKER_SETUP.md

    **Hot-reload section:**
    - Code changes in platform/backend or platform/frontend automatically reload
    - Volumes mounted from host
    - No need to rebuild images

    **Services section:**
    - postgres: Database (persistent volume)
    - redis: Message queue (future Bull queue support)
    - backend: Express API on port 8888
    - frontend: Next.js UI on port 3000

    **Useful commands section:**
    - View logs: `docker compose logs -f backend`
    - Restart service: `docker compose restart backend`
    - Stop all: `docker compose down`
    - Rebuild service: `docker compose up -d --build backend`
    - Run backend command: `docker compose exec backend npm test`

    **Troubleshooting section:**
    - Port conflicts: Check if ports 3000/8888/5432/6379 are in use
    - Hot-reload not working: Ensure CHOKIDAR_USEPOLLING=true (set in compose)
    - Database connection issues: `docker compose logs postgres`

    **Next steps:**
    - For worker setup and local Docker clankers, see LOCAL_DOCKER_SETUP.md
    - For AWS ECS deployment, see AWS_ECS_SETUP.md

    Keep it concise (target ~100 lines). Focus on the happy path.
  </action>
