  <action>
    Update .github/workflows/deploy-frontend-dev.yml to read Amplify configuration from SSM:

    1. Replace the "Start Amplify deployment" step content (lines 62-96):

       Current: Uses secrets.DEV_AMPLIFY_APP_ID and secrets.DEV_AMPLIFY_BRANCH
       New: Read from SSM parameters provisioned by Pulumi

    2. Update "Start Amplify deployment" step to:
       ```yaml
       - name: Start Amplify deployment
         id: amplify
         run: |
           # Get Amplify configuration from SSM (provisioned by Pulumi)
           APP_ID=$(aws ssm get-parameter \
             --name "/viberator/dev/amplify/appId" \
             --query Parameter.Value \
             --output text)

           BRANCH_NAME=$(aws ssm get-parameter \
             --name "/viberator/dev/amplify/branchName" \
             --query Parameter.Value \
             --output text)

           REGION=$(aws ssm get-parameter \
             --name "/viberator/dev/amplify/region" \
             --query Parameter.Value \
             --output text)

           if [ -z "$APP_ID" ]; then
             echo "::warning::Amplify app ID not found in SSM. Run pulumi up first."
             echo "deploymentTriggered=false" >> $GITHUB_OUTPUT
             echo "appUrl=https://dev.example.com" >> $GITHUB_OUTPUT
             exit 0
           fi

           echo "Starting Amplify deployment for app: $APP_ID, branch: $BRANCH_NAME, region: $REGION"

           # Trigger Amplify deployment using AWS CLI
           DEPLOYMENT_ID=$(aws amplify start-deployment \
             --app-id "$APP_ID" \
             --branch-name "$BRANCH_NAME" \
             --region "$REGION" \
             --source-url "https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }}" \
             --query 'jobSummary.id' \
             --output text)

           echo "Deployment started with ID: $DEPLOYMENT_ID"
           echo "deploymentId=$DEPLOYMENT_ID" >> $GITHUB_OUTPUT
           echo "deploymentTriggered=true" >> $GITHUB_OUTPUT
           echo "amplifyRegion=$REGION" >> $GITHUB_OUTPUT

           # Get the app URL for environment linking
           APP_URL=$(aws amplify get-app \
             --app-id "$APP_ID" \
             --region "$REGION" \
             --query 'app.defaultDomain' \
             --output text)

           echo "appUrl=https://$APP_URL" >> $GITHUB_OUTPUT
       ```

    3. Update "Wait for Amplify deployment to complete" step to use SSM for region:
       - Replace static APP_ID retrieval with SSM call (same as above)
       - Add REGION retrieval from SSM
       - Use --region "$REGION" in aws amplify commands

    4. Remove or update the comments about secrets - the workflow now uses SSM instead

    CRITICAL: All aws amplify commands must include --region "$REGION" since region is now dynamic.
  </action>

       Creates IAM OpenID Connect provider and role for GitHub Actions:

       - Permissions for Amplify deployment and SSM parameter reading
