  <name>Create Amplify OIDC component for GitHub Actions authentication</name>
  <files>infrastructure/components/amplify-oidc.ts</files>
  <action>
    Create infrastructure/components/amplify-oidc.ts with:

    1. Import statements for @pulumi/pulumi, @pulumi/aws, and InfrastructureConfig from ../config

    2. Export interface AmplifyOidcOptions:
       - config: InfrastructureConfig
       - githubRepository: string (e.g., "username/repo")

    3. Export interface AmplifyOidcOutputs:
       - providerArn: pulumi.Output<string>
       - roleArn: pulumi.Output<string>
       - roleName: pulumi.Output<string>

    4. Export function createAmplifyOidc(options: AmplifyOidcOptions): AmplifyOidcOutputs

    5. Inside createAmplifyOidc:
       a. Create aws.iam.OpenIdConnectProvider named "{environment}-viberator-github-oidc"
          - url: "https://token.actions.githubusercontent.com"
          - clientIdList: ["sts.amazonaws.com"]
          - thumbprintList: ["6938fd4d98bab03faadb97b34396831e3780aea1"]

       b. Create aws.iam.Role named "{environment}-viberator-amplify-github-actions-role"
          - assumeRolePolicy with OIDC trust:
            * Principal.Federated: oidcProvider.arn
            * Action: sts:AssumeRoleWithWebIdentity
            * Condition.StringEquals: token.actions.githubusercontent.com:aud = "sts.amazonaws.com"
            * Condition.StringLike: token.actions.githubusercontent.com:sub = "repo:{githubRepository}:*"
          - tags from config

       c. Create aws.iam.RolePolicy for Amplify permissions
          - Allow: amplify:StartDeployment, amplify:GetApp, amplify:GetJob, ssm:GetParameter
          - Resource: "*"

       d. Return providerArn, roleArn, roleName

    Follow the existing component pattern from registry.ts (options interface, outputs interface, factory function).

    NOTE: Project already has GitHub OIDC for Pulumi. This creates a separate provider/role for Amplify deployment to maintain separation of concerns.

<task type="auto">
  <name>Create Amplify frontend component for app and branch provisioning</name>
  <files>infrastructure/components/amplify-frontend.ts</files>
  <action>
    Create infrastructure/components/amplify-frontend.ts with:

    1. Import statements for @pulumi/pulumi, @pulumi/aws, and InfrastructureConfig from ../config

    2. Export interface AmplifyFrontendOptions:
       - config: InfrastructureConfig
       - backendUrl: pulumi.Input<string> (from load balancer output)
       - branchName?: string (default: environment === "prod" ? "production" : "main")
       - framework?: string (default: "Next.js - SSR")
       - stage?: string (default: environment === "prod" ? "PRODUCTION" : "DEVELOPMENT")

    3. Export interface AmplifyFrontendOutputs:
       - appId: pulumi.Output<string>
       - appArn: pulumi.Output<string>
       - defaultDomain: pulumi.Output<string>
       - branchName: string
       - ssmAppIdPath: string
       - ssmBranchNamePath: string
       - ssmRegionPath: string

    4. Export function createAmplifyFrontend(options: AmplifyFrontendOptions): AmplifyFrontendOutputs

    5. Inside createAmplifyFrontend:
       a. Determine branchName and stage from options/environment defaults

       b. Create aws.amplify.App named "{environment}-viberator-frontend"
          - name: "{environment}-viberator-frontend"
          - description: "Viberator frontend - {environment} environment"
          - platform: "WEB_COMPUTE" (CRITICAL - required for SSR)
          - buildSpec with:
            * backend.phases.preBuild.commands: ["npm ci --legacy-peer-deps"]
            * backend.phases.build.commands: ["npm run build"]
            * backend.artifacts.baseDirectory: ".next"
            * backend.artifacts.files: ["**/*"]
            * backend.cache.paths: ["node_modules/**/*"]
          - environmentVariables: { NEXT_PUBLIC_API_URL: options.backendUrl }
          - enableAutoBranchCreation: false (security)
          - tags from config

       c. Create aws.amplify.Branch named "{environment}-viberator-frontend-branch"
          - appId: app.id
          - branchName: branchName
          - stage: stage
          - framework: options.framework || "Next.js - SSR"
          - enableAutoBuild: false (build via GitHub Actions)
          - environmentVariables: { NEXT_PUBLIC_API_URL: options.backendUrl }
          - tags from config

       d. Create aws.ssm.Parameter for app ID
          - name: "/viberator/{environment}/amplify/appId"
          - value: app.id
          - type: "String"

       e. Create aws.ssm.Parameter for branch name
          - name: "/viberator/{environment}/amplify/branchName"
          - value: branchName (string literal)
          - type: "String"

       f. Create aws.ssm.Parameter for region
          - name: "/viberator/{environment}/amplify/region"
          - value: config.awsRegion
          - type: "String"

       g. Return outputs including SSM parameter paths

    Follow the existing component pattern. Use pulumi.interpolate for dynamic values like app.id in SSM values.

    CRITICAL: platform must be "WEB_COMPUTE" not "WEB" for SSR support.
    CRITICAL: enableAutoBranchCreation must be false for security.
  </action>
  <verify>

    - aws.amplify.Branch resource with enableAutoBuild: false

    - SSM parameter paths follow /viberator/{environment}/amplify/* pattern
  </verify>

  </done>
