<task type="auto">
  <name>Task 2: Create HeartbeatSweeper service</name>
  <files>platform/backend/src/workers/HeartbeatSweeper.ts</files>
  <action>
    Create HeartbeatSweeper class following the OrphanSweeper pattern:

    - Config interface: { sweepIntervalMs?: number; gracePeriodMs?: number }
    - Default: sweepIntervalMs = 60_000 (1 minute), gracePeriodMs = 300_000 (5 minutes)
    - Private properties: intervalId, jobService, config
    - Constructor sets defaults from config parameter
    - start() method: runs sweep() immediately, then setInterval
    - stop() method: clears interval
    - async sweep() method:
      1. Calculate staleThreshold = Date.now() - gracePeriodMs
      2. Find active jobs with last_heartbeat < staleThreshold OR last_heartbeat IS NULL AND started_at < staleThreshold
      3. For each stale job: updateJobStatus(jobId, 'failed', { errorMessage: 'Job failed: No heartbeat received within grace period' })
      4. Log warnings for each failed job
      5. Return count of jobs marked failed

    Add findStaleJobs() method to JobService:
    - SELECT id, started_at, last_heartbeat FROM jobs WHERE status = 'active' AND (last_heartbeat < $1 OR (last_heartbeat IS NULL AND started_at < $1))

    Console log format: [HeartbeatSweeper] prefix for all logs.
  </action>
  <verify>
    Check that HeartbeatSweeper follows OrphanSweeper pattern.
    Verify JobService.findStaleJobs() method exists.
    Verify conditional for NULL last_heartbeat handles jobs that never sent progress.
  </verify>
  <done>
    HeartbeatSweeper class exists with start/stop/sweep methods.
    JobService.findStaleJobs() query handles NULL last_heartbeat.
  </done>
</task>
