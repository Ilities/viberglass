**WebhookService class:**
Constructor dependencies (injected for testability):
- private registry: ProviderRegistry
- private configDAO: WebhookConfigDAO

- private jobService: JobService

Methods:

**async processWebhook(headers: Record<string, string>, payload: unknown, rawBody: Buffer): Promise<WebhookProcessingResult>**
Main orchestration method:

1. Route to provider:
   - const provider = this.registry.getProviderForHeaders(headers)
   - If no provider, return { status: 'ignored', reason: 'No matching provider' }

2. Parse event:
   - const event = provider.parseEvent(payload, headers)
   - Extract provider, eventType, deduplicationId, metadata

3. Resolve config:
   - Lookup config by metadata.repositoryId or metadata.projectId
   - If no config found, return { status: 'ignored', reason: 'No configuration found' }

4. Verify signature:
   - const secret = await this.secretService.getSecret(config)
   - const signature = headers[signatureHeader]
   - if (!provider.verifySignature(rawBody, signature, secret)):
       return { status: 'rejected', reason: 'Invalid signature' }

5. Check deduplication:
   - const { shouldProcess, existingId } = await this.deduplication.shouldProcessDelivery(event.deduplicationId)
   - if (!shouldProcess):
       return { status: 'duplicate', existingId }

6. Record delivery start:
   - const deliveryId = await this.deduplication.recordDeliveryStart({ provider, deliveryId: event.deduplicationId, eventType: event.eventType, payload })

7. Process event (provider-specific):
   - await this.processProviderEvent(event, config, deliveryId)

8. Record success:
   - await this.deduplication.recordDeliverySuccess(event.deduplicationId, ticketId, projectId)

**private async processProviderEvent(event: ParsedWebhookEvent, config: WebhookConfig, deliveryId: string): Promise<void>**
Provider-specific event handling:

For GitHub 'issues' event with action 'opened':
- Extract issue data from event.payload
- Create ticket via ticketDAO.createTicket()
- Map GitHub issue fields to ticket fields (title, description, metadata)
- Store external_ticket_id (issue number) and external_ticket_url
- If config.auto_execute: Create job via jobService

For GitHub 'issue_comment' event:
- Check if comment mentions bot (@config.bot_username)
- If mentioned and contains keywords (e.g., 'fix this'): Create ticket and optionally job

**async getFailedDeliveries(limit?: number): Promise<WebhookDeliveryAttempt[]>**
- Delegate to this.deduplication.getFailedDeliveries(limit)

**async retryDelivery(deliveryId: string): Promise<WebhookProcessingResult>**
- Fetch delivery record
- Re-process through processWebhook flow

Define interfaces:
- WebhookProcessingResult: { status: 'processed' | 'ignored' | 'rejected' | 'duplicate' | 'failed', ticketId?, jobId?, reason?, existingId? }

  <action>
Refactor existing webhooks.ts to use the new provider architecture.

**Remove:**
- Old verifyGitHubSignature middleware (use new createSignatureMiddleware)
- Direct GitHub integration instantiation
- Inline webhook event processing

**Add:**
- Import from webhooks module:
  - ProviderRegistry, GitHubWebhookProvider
  - WebhookService, WebhookSecretService
  - createSignatureMiddleware, rawBodyMiddleware
  - WebhookConfigDAO, WebhookDeliveryDAO, DeduplicationService

- Initialize providers and services:
  ```typescript
  const registry = new ProviderRegistry();
  const githubProvider = new GitHubWebhookProvider();
  registry.register(githubProvider);

  const configDAO = new WebhookConfigDAO(db);
  const deliveryDAO = new WebhookDeliveryDAO(db);
  const secretService = new WebhookSecretService(credentialProvider);
  const deduplication = new DeduplicationService(deliveryDAO);
  const webhookService = new WebhookService(
    registry, configDAO, deliveryDAO, deduplication,
    secretService, ticketDAO, jobService
  );
  ```

**Update routes:**

POST /api/webhooks/github:
- Use rawBodyMiddleware
- Use createSignatureMiddleware with getSecret function
- Call webhookService.processWebhook(headers, req.body, req.rawBody)
- Return appropriate response based on result status

POST /api/webhooks/jira:
- Placeholder for Jira provider (to be implemented)
- Return 501 for now

GET /api/webhooks/status:
- Update to include failed deliveries count from deliveryDAO

POST /api/webhooks/:deliveryId/retry:
- New endpoint for manual retry
- Call webhookService.retryDelivery(deliveryId)
- Return processing result

GET /api/webhooks/configs:
- List all webhook configurations
- Use configDAO to fetch

POST /api/webhooks/configs:
- Create new webhook configuration
- Use configDAO.createConfig

DELETE /api/webhooks/configs/:id:
- Delete webhook configuration
- Use configDAO.deleteConfig
