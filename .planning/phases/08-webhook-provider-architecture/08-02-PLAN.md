<task type="auto">
  <name>Task 2: Create signature validator with timing-safe comparison</name>
  <files>platform/backend/src/webhooks/validators.ts</files>
  <action>
Create validators.ts with SignatureValidator class following RESEARCH.md patterns.

**SignatureValidatorConfig interface:**
- algorithm: 'sha256' | 'sha1'
- headerName: string (e.g., 'x-hub-signature-256')
- prefix: string (e.g., 'sha256=')

**SignatureValidator class:**
- constructor(private config: SignatureValidatorConfig)
- verify(payload: Buffer, signature: string, secret: string): boolean

**verify implementation:**
1. Strip prefix from signature if present
2. Create HMAC using crypto.createHmac(algorithm, secret)
3. Update with payload bytes
4. Get digest as hex
5. Convert both signatures to Buffer
6. Check buffer lengths match
7. Use crypto.timingSafeEqual() for comparison

**CRITICAL:** Use crypto.timingSafeEqual() - NOT === or == - to prevent timing attacks (per GitHub docs).

Export SignatureValidator and SignatureValidatorConfig.
</action>
  <verify>
1. SignatureValidator uses crypto.timingSafeEqual() for comparison
2. Algorithm defaults to sha256 for GitHub
3. Prefix stripping handles both 'sha256=' and raw hex formats
4. Returns false for buffer length mismatch before comparison
5. Run: npm run build (in platform/backend) compiles
</verify>
  <done>
SignatureValidator provides secure, timing-safe HMAC verification for webhook signatures.
</done>

Create two middleware files in platform/backend/src/webhooks/middleware/.

**rawBody.ts:**
Export rawBodyMiddleware() using express.raw({ type: 'application/json', limit: '10mb' }).
This captures raw request body BEFORE JSON parsing (required for signature verification).

**signature.ts:**
Create createSignatureMiddleware(validator: SignatureValidator, getSecret: (projectId?: string) => Promise<string>)

ExtendedRequest interface extends Request with rawBody?: Buffer.

Middleware flow:
1. Extract signature from configured header (x-hub-signature-256, x-hub-signature)
2. If missing, return 401
3. Get rawBody from request (set by rawBodyMiddleware)
4. Get secret using provided getSecret function
5. Call validator.verify(rawBody, signature, secret)
6. If invalid, return 401
7. Parse JSON manually: req.body = JSON.parse(rawBody.toString('utf8'))
8. Call next()

IMPORTANT: This middleware must be applied AFTER rawBodyMiddleware and BEFORE any JSON parsing middleware.

Export: { createSignatureMiddleware, ExtendedRequest }
</action>

2. Signature middleware checks for signature header presence
3. Signature is verified against rawBody, not parsed JSON
4. req.body is replaced with parsed JSON after verification
5. Returns 401 for missing or invalid signatures
6. Run: npm run build (in platform/backend) compiles
</verify>
  <done>
Middleware chain captures raw body, verifies signature securely, then parses JSON for downstream handlers.
