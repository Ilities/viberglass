  <action>
Create migration file with Kysely up() and down() functions.

**Table: webhook_provider_configs**
- id (varchar(255), primary key) - UUID
- project_id (varchar(255), foreign key to projects.id, nullable, unique)
- provider (varchar(50), not null) - 'github' or 'jira'
- provider_project_id (varchar(255), nullable) - External project ID (e.g., repo full name for GitHub, project key for Jira)
- secret_location (varchar(20), not null, default 'database') - 'database', 'ssm', or 'env'
- secret_path (varchar(500), nullable) - SSM parameter path or env var name
- webhook_secret_encrypted (text, nullable) - Encrypted webhook secret for database storage
- allowed_events (jsonb, default '[]') - Array of event types to process
- auto_execute (boolean, default false) - Auto-create job after ticket creation
- bot_username (varchar(100), nullable) - For @mention detection (e.g., 'viberator-bot')
- label_mappings (jsonb, default '{}') - Map labels to clanker slugs
- active (boolean, default true) - Enable/disable webhook processing
- created_at (timestamp, default CURRENT_TIMESTAMP)
- updated_at (timestamp, default CURRENT_TIMESTAMP)

Unique index on (project_id) since one provider config per project.

**Table: webhook_delivery_attempts**
- id (varchar(255), primary key) - UUID
- provider (varchar(50), not null) - 'github' or 'jira'
- delivery_id (varchar(255), not null, unique) - Provider's delivery ID for deduplication
- event_type (varchar(100), not null) - Event type from webhook
- status (varchar(20), not null) - 'pending', 'processing', 'succeeded', 'failed'
- error_message (text, nullable)
- payload (jsonb, not null) - Original webhook payload
- project_id (varchar(255), nullable) - Associated project if resolved
- ticket_id (varchar(255), nullable) - Created ticket ID if applicable
- created_at (timestamp, default CURRENT_TIMESTAMP)
- processed_at (timestamp, nullable)

Indexes:
- idx_webhook_delivery_attempts_delivery_id on (delivery_id)
- idx_webhook_delivery_attempts_status on (status)
- idx_webhook_delivery_attempts_created_at on (created_at DESC)

Use migration pattern from 005_add_jobs_table.ts and 007_add_job_heartbeat_and_progress.ts.
