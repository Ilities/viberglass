  <name>Task 1: Create webhook API service</name>
  <files>platform/frontend/src/service/webhook.ts</files>
  <action>
Create webhook.ts service file following the pattern of existing services (clanker.ts, data.ts).

**WebhookService class with static methods:**

- static async getWebhookConfigs(projectId?: string): Promise<WebhookConfig[]>
  - GET /api/webhooks/configs?projectId={projectId}
  - Return array of configurations

- static async createWebhookConfig(config: CreateWebhookConfigDTO): Promise<WebhookConfig>
  - POST /api/webhooks/configs
  - Return created configuration

- static async updateWebhookConfig(id: string, config: UpdateWebhookConfigDTO): Promise<WebhookConfig>
  - PUT /api/webhooks/configs/{id}
  - Return updated configuration

- static async deleteWebhookConfig(id: string): Promise<void>
  - DELETE /api/webhooks/configs/{id}

- static async testWebhookConfig(configId: string): Promise<TestResult>
  - POST /api/webhooks/configs/{configId}/test
  - Send synthetic test event
  - Return { success: boolean, message: string }

- static async getFailedDeliveries(limit?: number): Promise<WebhookDelivery[]>
  - GET /api/webhooks/deliveries?status=failed&limit={limit}
  - Return failed deliveries

- static async retryDelivery(deliveryId: string): Promise<RetryResult>
  - POST /api/webhooks/deliveries/{deliveryId}/retry
  - Return { success: boolean, ticketId?, jobId?, message }

- static async getDeliveryStatus(): Promise<DeliveryStatus>
  - GET /api/webhooks/status
  - Return webhook processing statistics

**TypeScript interfaces:**
- WebhookConfig: id, projectId, provider, providerProjectId, secretLocation, allowedEvents, autoExecute, botUsername, labelMappings, active, createdAt, updatedAt
- CreateWebhookConfigDTO: Same as WebhookConfig without id, createdAt, updatedAt
- UpdateWebhookConfigDTO: Partial of CreateWebhookConfigDTO
- WebhookDelivery: id, provider, deliveryId, eventType, status, errorMessage, createdAt, processedAt
- TestResult, RetryResult, DeliveryStatus as appropriate

Use fetch with error handling matching existing service patterns.
